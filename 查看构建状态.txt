========================================
查看 GitHub Actions 构建状态
========================================

方法 1：访问 Actions 页面
--------------------
直接访问：
https://github.com/wwwgzpucom/qjmarriage-ios-app/actions

你会看到：
- 所有构建任务列表
- 每个任务的状态（✅ 成功 / ❌ 失败 / 🟡 进行中）
- 构建时间


方法 2：查看最新构建
--------------------
1. 访问仓库主页：
   https://github.com/wwwgzpucom/qjmarriage-ios-app

2. 点击顶部的 "Actions" 标签

3. 查看最新的 "iOS Build" 工作流


方法 3：查看 README 徽章
--------------------
README 顶部有构建状态徽章：
- 绿色 ✅ = 构建成功
- 红色 ❌ = 构建失败
- 黄色 🟡 = 正在构建


========================================
构建时间说明
========================================

首次构建通常需要：
- 5-10 分钟（正常情况）
- 可能更长（如果 GitHub 服务器繁忙）

构建步骤：
1. 检出代码 (10秒)
2. 设置 Node.js (20秒)
3. 安装依赖 (1-2分钟)
4. 构建 Web 资源 (30秒)
5. 同步到 iOS (30秒)
6. 安装 CocoaPods (1-2分钟)
7. 构建 iOS 应用 (2-3分钟)
8. 上传构建产物 (30秒)


========================================
如果构建失败
========================================

1. 点击失败的构建任务
2. 查看红色 ❌ 的步骤
3. 点击展开查看错误日志
4. 常见问题：
   - 依赖安装失败 → 网络问题，重新运行
   - CocoaPods 失败 → 版本问题
   - Xcode 构建失败 → 代码问题


========================================
下载构建产物
========================================

构建成功后：

1. 进入成功的构建任务
2. 滚动到页面底部
3. 找到 "Artifacts" 部分
4. 下载 "ios-build.zip"
5. 解压后可以在模拟器中测试


========================================
手动触发构建
========================================

如果想重新构建：

1. 访问 Actions 页面
2. 点击 "iOS Build" 工作流
3. 点击右侧 "Run workflow" 按钮
4. 选择 "main" 分支
5. 点击绿色 "Run workflow" 按钮
